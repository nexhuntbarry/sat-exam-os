## 🟢 Quick wins (do first)

### 5. Admin can see students' test results from teacher section
**Get it.** Admin should be able to see every test's results page even though they're not the assigned teacher.
- Scope: **S**
- Today: `/teacher/tests/[id]/results` requires teacher_ids includes user — admin role isn't auto-allowed.
- Fix: relax auth so admin role bypasses the assignment check. Same for analytics page.
- Q: should admin see results in `/admin/...` namespace too, or just have admin role automatically work in `/teacher/...`?

### 6. Admin can see each student's results from students section
**Get it.** `/admin/students/[id]` should list every test that student took + scores + drill-in.
- Scope: **S**
- New section on student detail page: "Tests taken" table (test name / submitted at / score / time spent / link).
- Reuses existing `submissions` data.

### 10. Auto-end test when time is up
**Get it.** TestTimer expires → auto-submit, no human click needed.
- Scope: **S**
- Today: TestTimer probably calls `onExpire` which calls submit. Need verify that path actually finalizes the submission instead of just locking UI.
- Q: when timer expires, do you want partial answers auto-saved + status="Submitted", or "Auto-Submitted" (separate status)?

### 12. Show "Blank" instead of just "answered/unanswered"
**Get it.** In results review, each unanswered question should show "Blank" (not blank cell or "—").
- Scope: **S**
- Cosmetic + tooltip-friendly. Trivial.

### 15. Invite more students to a test that's already created
**Get it.** Add students to an existing test_assignment without recreating.
- Scope: **S-M**
- New endpoint `POST /api/admin/tests/[id]/add-students` + UI on test detail.
- Q: if test is already published and someone else submitted, do new students see the same questions / time limit? Yes I assume.

### 16. Admin sees when + which tests each student took
**Get it.** Same as #6 — student detail page shows history. Combine with #6.
- Scope: **S** (covered by #6).

## 🟠 Bigger features (need design first)

### 1. Choose 2 modules when creating a test
**Get it.** Bundle Math Module 1 + Math Module 2 (or R&W M1 + R&W M2) into one test, since real SAT is adaptive across two modules.
- Scope: **M**
- Schema today: `tests.module_id` is a single FK. Need either (a) `module_ids` array or (b) `tests.section_module_1` + `section_module_2`.
- Q: do you want adaptive routing (Module 2 difficulty depends on Module 1 score, like real SAT)? Or just sequential (always serve M1 then M2 regardless of score)?
- Q: timer is per-module (32 min each on real SAT) or combined?

### 3. Desmos calculator + formula sheet dropdown when taking test
**Get it.** Math section: embed Desmos graphing calculator + a dropdown showing common formulas (admin curated).
- Scope: **M-L**
- Desmos has free embed: `https://www.desmos.com/calculator?embed`. Loads in iframe. Easy.
- Formula sheet: new admin-managed table `formula_sheets` with markdown content + section assignment. Drop-down on test-taking UI.
- Q: only Math section sees these, right? R&W shouldn't.
- Q: should formula content per-test (admin picks which sheet) or global default?

### 14. SAT score calculator (200-800 per section, keep percentage too)
**Get it.** Show raw correct count, percentage AND scaled SAT score (200-800 per section, 400-1600 total).
- Scope: **M-L**
- Real SAT scoring uses adaptive scaling tables that depend on which Module 2 difficulty was served. Without adaptive logic, we approximate via static lookup table: e.g. R&W 27/27 = 800, 20/27 ≈ 700, etc.
- Q: do you have an official scaling table to use? College Board publishes one per practice test.
- Q: for now use a generic approximation table, or wait until you can supply the real one?

## 🔴 Pedagogy features (most ambitious)

### 4. Annotation + highlight for R&W during test
**Get it.** Student highlights text + writes margin notes in R&W passages while taking the test.
- Scope: **M-L**
- Highlights persist to `submissions.metadata.annotations[questionId]` JSON.
- Library: don't need a heavy editor — text-selection highlight + small popover for color/note.
- Q: do annotations save per test session only, or persist across retakes?
- Q: visible to teacher in review? My guess: yes — useful for tutoring.

### 8 + 17. Review mode — teacher reveals questions / answers in class
**Get it.** Teacher has a "Reveal" button. When on, students in that class can see test questions + correct answers (post-test review).
- Scope: **M-L** (combined)
- Schema: `tests.review_mode_enabled BOOLEAN` or `test_assignments.review_unlocked_for: ["studentId"...]`.
- UI: teacher review-mode page lets them step through questions live; students' devices show same question synced (or just unlocked).
- Q: synchronous reveal (every student sees question N when teacher is on N) or async (students can re-open test in read-only mode any time after teacher unlocks)?
- Q: per-student unlock or all-class unlock?

### 13. Assign groups/students to teacher, teacher reviews their results separately
**Get it.** Persistent teacher↔student mapping (not just per-test). Teacher's "My Students" page shows everyone they're responsible for, with results across all tests.
- Scope: **M**
- Today: assignment is per-test via `test_assignments.teacher_ids`. Need `teacher_assignments` table linking teacher → student or teacher → class_group.
- Q: assign by class_group (cleaner, scales) or by individual student? My recommendation: by class_group.
- Q: can a student be assigned to multiple teachers (e.g., main + 1-on-1 tutor)?
